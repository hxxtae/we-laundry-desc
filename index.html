WeLaundry

Laundry customized POS web application.

Description

  • 기존 서비스업, 외식업에서 사용되는 포스를 바탕으로 제작한 세탁소 전용 포스 웹 애플리케이션 입니다.

  • Why?

    '포스(POS)'라는 키워드를 선택한 이유는 당시 부모님께서 처음으로 세탁소 운영을 시작 하셨을 때 이었습니다.
    돈을 지불하고 세탁업종 포스기를 사용하는 것 보단 부모님의 의견을 반영한 포스 프로그램을 사용하게 해 드리고 싶었습니다. 그리고 대중적인 포스 시스템은 일반 음식점이나, 카페, 의류매장에서 많이 사용하지만, 세탁소 업종에서는 포스 시스템이 많지 않거나, 수기로 장부를 사용하기 때문에 이를 개선하고자 프로젝트를 기획하게 되었습니다.

  • How?

    우리동네 세탁소에서는 하루, 일주일에 수백가지 옷 개수와 종류가 세탁소에 맡겨집니다. 하나하나 장부 책자를 넘겨가면서 찾을 필요 없이 전자 장부처럼 활용 할 수 있으며, 더 나아가 날짜별, 고객별 등 데이터를 손쉽게 조회하고, 나의 매출, 손님 리스트, 품목 관리 등을 활용하여 세탁소 운영에도 도움을 얻을 수 있습니다.

  • Who?

    큰 기업의 세탁소가 아닌 일반 세탁소(소상공인)를 대상으로 합니다.

Login/Signup (로그인/회원가입)

Before

⭕ 회원가입 페이지 및 기능 미포함.

⭕ 로그인 시 서버 인증 방식(세션 & 쿠키)을 통한 사용자 접속 식별 및 유지.

After

✔ 회원가입 페이지 및 기능 포함.

✔ 회원가입 시 기존 사용자 중복확인 및 메세지 출력.

Main (메인화면)

Before

⭕ 최초 로그인 접속 시 에만 출력되는 페이지.

⭕ 동적 SVG 이미지 베너 이외 다른 기능은 존재하지 않음.

After

✔ 최초 로그인 및 토큰(브라우저 쿠키) 존재 시 최초 접속 후 출력되는 페이지.

✔ "포스 애플리케이션" 운영(영업) 시작 및 종료 기능 추가.

✔ 영업 시작 시 시간 기록 및 Order(주문접수) 페이지 활성화.

Address (주소관리)

Before

⭕ 정확한 우편번호(주소)를 찾을 수 있는 검색 기능이 없다.

⭕ 주소ID는 고유번호를 오름차순으로 생성하여 부여하는 방식으로 관리됨. (Oracle의 Sequence 사용)

After

✔ 정확한 우편번호(주소)를 찾을 수 있는 우편번호 오픈소스 를 사용.

✔ 주소ID는 중복이 없는 고유번호를 생성하는 방식으로 변경. (MongoDB의 ObjectId 사용)

✔ 많은 주소 데이터 관리를 위해 페이징 처리 됨.

Customer (고객관리)

Before

⭕ 고객 정보 생성 시 주소(단지이름, 동, 호) 는 Require, 전화번호는 Option

⭕ 고객 고유번호는 DB의 Sequence로 생성

⭕ 고객 조회는 고객번호, 전화번호, 주소(단지이름, 동, 호)로 조회

After

✔ 고객 정보 생성 시 단지이름, 동, 호 만 Require

✔ 고객 ID는 DB의 Unique ObjectId로 생성

✔ 고객 조회는 주소(단지이름, 동, 호)로 조회

Products (품목관리)

Before

⭕ 카테고리별 품목 리스트의 추가, 수정, 삭제 가능.

⭕ 직관적이지만 버튼이 많아 난잡한 UI로 가독성이 떨어진다.

After

✔ 품목 CRUD 뿐만 아니라 카테고리 추가, 변경, 삭제 가능.

✔ 품목 이란 많은 항목을 관리하기 때문에 최대한 간결하게 구성.

✔ 카테고리 내에서 원하는 품목 순서 배치가 가능.

✔ 폼목 관리를 위한 설정버튼 구성.

Order (주문접수)

Before

⭕ 단지이름, 동, 호 수로 검색 이후 고객을 선택하여 데이터 Set.

⭕ 품목 찾기 버튼 클릭 시, 폼목 팜업에서 선택하여 주문 데이터 Set.

⭕ 다이나믹한 주문내역 수정이 불가능.

⭕ 주문내역 영수증 출력.

After

✔ 단지이름, 동, 호 만 가지고 바로 고객 데이터 Set.

✔ 품목 박스 클릭 만으로 주문 데이터 Set.

✔ 사용자가 원하는 대로 주문내역 수정이 가능.

✔ 접수 진행 시 주문내역확인 및 결제진행, 할인율 적용 화면 추가.

History (주문내역)

⭕ 주문번호, 날짜, 주소, 전화번호 로 주문내역 검색.

⭕ 주문내역 선택 시 팝업에서 주문상세내역 확인 가능.

⭕ 주문내역 선택 시 영수증 출력 가능.

After

✔ 날짜, 주소로 주문내역 검색.

✔ 주문내역 선택 시 주문상세내역 확인 가능.
